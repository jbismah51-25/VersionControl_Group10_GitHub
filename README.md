##  Purpose
This repository is created for **Assignment ** of *Software Quality Engineering* to practice version control using **GitHub**.  
The goal is to learn how to:
- Create and manage repositories  
- Use branching, committing, and merging  
- Resolve conflicts  
- Collaborate effectively on shared projects  

---

##  Tasks and Workflow
1. Team leader (Group 10 – GitHub) creates this repository and adds all collaborators.  
2. Upload an initial document: `ProjectReport.docx` (describing GitHub and its key features).  
3. Each collaborator will:
   - Create their own branch from the main branch.  
   - Make **three commits** (Introduction, Details, Conclusion).  
   - Push their branch to the repository.  
4. The team leader will:
   - Review and merge all branches.  
   - Resolve any conflicts if they occur.  
   - Update this README with final remarks.
